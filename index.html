Lets start over.

What if we just have some generic structures instead?

They could have some contextually updated properties, dependant on everyone else.

Let's call this their context

They could have bits of data that everyone else is also concerned with.

Let's call this their data

They could have some configuration that only they are concerned with.

Let's call this their config

While we're naming things, let's call this monitored data/context thing a DataContext.

And DataContexts need to live in some kind of group so they can keep track of each other. Let's call that a ContextGroup

Ok, awesome. So here's roughly what we've got: 
var DataContext = {
    data: {}, // Things that affect everything else
    config: {}, // Things that only affect this one
    context: {} // The contextual bit
}
var ContextGroup = {
    items: [DataContext, DataContext, ...]
}

Let's define some simple rules
When any data changes on a DataContext, everyone else needs to update
When any config changes, only that DataContext needs to update
So how might this work with our crazy UI?
Let's look at query 
var query = {
    data: {
        search: 'someSearch'
    }
    // No config and no context
}
What about results? 
var results = {
    // No data
    config: {
        sort: 'PO.IssuedDate',
        page: 1,
        ...
    },
    context: {
        records: [PurchaseOrder, PurchaseOrder, ..]
    }
}
Checkbox lists (facet)? 
var facet = {
    data: {
        checkedValues: ['a', 'b', ...],
        mode: 'include'
    },
    config: {
        count: 10,
        find: 'someString'
    },
    context: {
        options: [{ name: 'example', count: 123 }, ...]
    }
}
Even... charts! 
var someChart = {
    // No data
    config: {
        // key and value fields, counts, etc
    },
    context: {
        // top terms, etc
    }
}
Any other event types besides dataChange and configChange?
Of course :)
So we react to events with these things called Reactors. Great name, right?
dataChange has an others reactor
configChange has an only reactor
Others for things like typeChange, itemAdded, itemRemoved, etc
So what exactly is a reactor? Just a function: 
function reactor(items, instigator) {
    // items is the list of DataContexts in the ContextGroup
    // instigator is the DataContext whose event this is about
    return arrayOfDataContextsThatNeedToUpdate;
}
